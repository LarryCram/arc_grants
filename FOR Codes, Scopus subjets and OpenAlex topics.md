this is true. A FOR has a code and a name. FOR codes are nested - the first 2 digits are the division and the 4 digits are the group. There is also a 6 digit code called the field that is not used in this project. There is a 2008 and a 2020 version of the FOR. The only place that the 2008 appear are FOR codes in grants awadred prior to the use of the 2020 FOR. There is a pre-created table of 2008 FOR to 2020 FOR that is applied when the ARC data is first extracted. Openalex topics are a hierachy, domain, field, subfield and topic. This is not encoded in a string but only at JSON level. The subfield level is deliberately mapped to the 254 Scopus minor subjects/subcategories/subfields, which are nested beneath 4 supergroups equal to oax domains, and 27 subjects/categories/fields equal to OAX fields. OAX fields do not include the muiltidicipline categores of Sopus - this diference relates to journal level and paper level topic coding. There is a bidirectional 1:1 mapping of FOR dividions to Scopus/OAX fields and of FOR groups to Scopus/OAX subfields.
